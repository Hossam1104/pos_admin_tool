Standard Operating Procedure (SOP)

1. Daily Monitoring

Check the Service Control panel at the start of your shift. Ensure all critical services (Branch, Cashier) are Running.

2. Database Backup

Perform backups before any major system changes or at the end of the day:

  1. Navigate to the Backup panel.
  2. Select the target databases.
  3. Click Execute Backup.
  4. Verify the success message in the System Log.

3. Database Restore

CRITICAL ACTION: Only restore when absolutely necessary (e.g., data corruption).

  1. Stop Services: Go to Service Control and stop Branch/Cashier services.
  2. Select Backup: Browse for the correct .bak file.
  3. Target: Ensure the "Client Name" matches the store context.
  4. Confirm: Click Restore and provide the administrative confirmation.

4. Emergency Recovery

If the system is unresponsive or services are stuck:

  • Go to the Danger Zone.
  • Type CONFIRM DANGER ZONE and click Execute.
  • This will force-stop all components and clear temporary blocks.

DBS | RMS+ POS Admin Tool

Production v1.0 | Built for Digital Business Systems (DBS) RMS+ POS Module

Key Features

  • Service Management: Real-time monitoring and control of RMS+ services
  • Database Backup/Restore: Automated backup with strict naming conventions
  • Safety Validation: Prevents operations on invalid paths or active databases
  • Visual Feedback: Color-coded status indicators and real-time logging
  • Configuration Persistence: JSON-based settings that persist between sessions

System Requirements

  • OS: Windows 10/11 or Windows Server 2016+
  • Database: Microsoft SQL Server 2014+
  • Privileges: Administrator elevation required
  • Python: 3.10+ (for source execution)
  • Dependencies: PySide6, pywin32

Architecture Overview

[ UI Layer (View) ] <---> [ MainController ] <---> [ Logic Layer (BatchRunner) ]
       |                          |                        |
[ ServiceMonitor ]        [ Config Manager ]       [ System / SQLCMD ]
       |                                                   |
[ Windows SCM ]                                     [ SQL Server ]

Safety Guidelines

  • Always Backup before attempting a Restore operation
  • Verify SQL permissions (sysadmin or dbcreator roles required)
  • Do not assume "Success" means data integrity - verify application loads after restore
  • Danger Zone operations require explicit confirmation
